Data

‘We Need Us’ uses classification and user data from Zooniverse.org. The data is accessed in real-time via an API (application programming interface). Each time a user logins and classifies a piece of data on any project in the Zooniverse, some metadata about that action is stored in a database and is published as a live feed. ‘We Need Us’ take the Zooniverse metadata and stores it in a useful format in a new database. The metadata is processed to create more metadata, which we call timeseries data. We store eight different types of timeseries data for the ‘We Need Us’ sound and animation engines to use, that are constantly updated:

  • min - the number of classifications per minute, over 60 minutes
  • min15 - the number of classifications per 15 minute intervals over 15 hours
  • hour - the number of classifications per hour over 24 hours
  • day - the number of classifications per day over 30 days
  • min - the number of users per minute, over 60 minutes
  • min15 - the number of users per 15 minute intervals over 15 hours
  • hour - the number of users per hour over 24 hours
  • day - the number of users per day over 30 days
